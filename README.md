# Project Kaggle — Classic ML & DL

## О проекте

Репозиторий содержит полные пайплайны решения двух классических задач:

| Соревнование | Задача | Целевая метрика |
|--------------|--------|-----------------|
| **Titanic** | Классификация (выживание пассажиров) | Accuracy |
| **House Prices** | Регрессия (предсказание цены дома) | RMSE |

В каждом проекте реализованы: EDA, предобработка, кросс-валидация, классические ML-модели, нейросети (PyTorch), feature engineering и ансамбли.

Перейдите в папку `Titanic` или `HousePrices` и открывайте ноутбуки по порядку (01 → 08).

## Пайплайн (для каждого соревнования)

1. **01_dataset_EDA** — разведочный анализ (статистики, распределения, визуализации).
2. **02_preprocessing** — работа с пропусками, выбросами, кодирование категориальных признаков, масштабирование.
3. **03_validation** — разбиение на фолды (Stratified K-Fold для Titanic, K-Fold для House Prices).
4. **04_models** — классические модели: линейная/логистическая регрессия, Ridge, Lasso, ElasticNet, KNN, дерево решений, Random Forest, XGBoost, LightGBM, CatBoost.
5. **05_DL_models** — нейросети на PyTorch.
6. **06_feature_engineering** — создание и отбор признаков.
7. **07_ensemble** — ансамбли (voting, averaging, stacking).
8. **08_Final_results_summary** — сводка всех результатов и лучших моделей.

## Установка и запуск

### Требования

- Python 3.10  
- Conda (рекомендуется) или совместимый менеджер окружений  

### Создание окружения

```bash
cd ProjectKaggle
conda env create -f environment.yml
conda activate ml_final
```

Если у вас нет GPU, в `environment.yml` можно закомментировать или удалить строку с `pytorch-cuda=11.8`.

Конфигурация (пути к данным, гиперпараметры моделей, настройки препроцессинга) задаётся в `config.py` внутри каждой папки проекта.
